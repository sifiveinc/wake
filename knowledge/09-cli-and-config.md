## Workspace setup

```bash
wake --init .          # create .wakeroot and wake.db here
```

A workspace is any directory containing `.wakeroot` (or any ancestor with one).

## Running things

```bash
wake <fn> [args...]              # invoke a Wake function `fn: List String -> _`
wake -x 'expr'                   # evaluate an expression
wake -xv 'identifier'            # also print its inferred type
wake --in <package> -x 'expr'    # evaluate inside a specific package's namespace
```

Wake auto-discovers `.wake` files under the workspace (modulo `.wakeignore`).

## Database introspection

Every job is recorded in `wake.db`. Useful queries:

| Flag | What it shows |
|---|---|
| `wake --last` | Most recent jobs |
| `wake --failed` | Jobs that failed |
| `wake --job <id>` | Full record for a job |
| `wake -o <path>` | Jobs that wrote `<path>` |
| `wake --output <path>` | Same as `-o` |
| `wake -i <path>` | Jobs that read `<path>` |
| `wake --input <path>` | Same as `-i` |
| `wake -g` | All global identifiers in scope |
| `wake --metadata <…>` | Metadata only (skip stdout/stderr) |

Combine with `-v` (verbose; include captured stdout/stderr) and `-d` (debug; include stack traces).

## Verbosity & log routing

Global flags:

| Flag | Effect |
|---|---|
| `-v`, `--verbose` | Show `info`, `echo`, `report`, `warning`, `error`. |
| `-d`, `--debug`   | Show debug + everything `--verbose` does. Adds stack traces. |
| `-q`, `--quiet`   | Show only `error`. |
| (default)         | Show `report`, `warning`, `error` (+ `interactive` if TTY). |

Custom routing (override defaults; comma-separated log levels):

```bash
wake --stdout=info,echo,report     ...     # send these to stdout
wake --stderr=warning,error       ...      # send these to stderr
wake --fd:5=debug                ...       # route a level to fd 5
```

Built-in `LogLevel`s (see `share/doc/wake/logging.md` and `share/wake/lib/core/print.wake`):
- `logDebug` ("debug")
- `logInfo` ("info")
- `logEcho` ("echo") — the command line being executed
- `logInteractive` ("interactive") — only if attached to a TTY
- `logReport` ("report") — `print` / `println` output
- `logWarning` ("warning")
- `logError` ("error")
- `logNever` ("null") — black hole

User code can declare new ones: `def myLog = LogLevel "diag"`.

## TTY / progress

- `--no-tty` — disable progress bar and TTY-only logger output.
- The progress UI uses ncurses on TTY environments.

## Configuration files

### `.wakeroot` (per-workspace)

JSON5 (so trailing commas and comments are fine). Empty file is also valid (workspace marker only).

```json5
{
  version: ">=49.0.0",          // SemVer constraint on wake binary
  user_config: ".wake/user.json",
  log_header: "[$source $stream] ",
  log_header_source_width: 24,
  log_header_align: true,
  max_cache_size: 25_000_000_000,    // 25 GB
  low_cache_size: 15_000_000_000,    // 15 GB
  cache_miss_on_failure: true,
}
```

### `~/.config/wake.json` (per-user)

Same keys as `.wakeroot`. Per-workspace `.wakeroot` overrides user defaults.

### `.wakemanifest`

Auto-generated by `make tarball`. Lists every `.wake` file shipped in the source tarball release. You shouldn't hand-edit it.

### `.wakeignore`

Glob patterns to exclude `.wake` files. Lives next to a `.wakeroot`. Syntax:

```
# comment
build/**            # ignore everything under build/
**/legacy.wake      # at any depth
?/[a-z]*.wake       # ranges & single-char wildcards
```

## fd 3 and fd 4 — runner channels

By convention:
- **fd 3** carries runner-specific *output* (structured metadata, e.g., observed inputs/outputs).
- **fd 4** carries runner-specific *errors*.

Job code can produce these via `unsafe_reportJobRunnerOutput` etc.; consumers use `unsafe_getJobRunnerOutput` / `unsafe_getJobRunnerError`. See `share/doc/wake/logging.md` for the full convention.

## Quick reference card

```
wake --init .                     # initialize workspace
wake -x '1+2'                     # evaluate expression
wake build default                # run the build via build.wake
wake --last                       # recent jobs
wake -o some/output.o             # which job produced this?
wake -i src/main.cpp              # which jobs read this?
wake --job 1234 -v                # full log of a job
wake -g | grep linkO              # find a stdlib symbol
wake --verbose <fn> args          # run with extra logging
```
